Quick Export

The Quick Export Plugin allows you to configure export settings for your images once and then export them with a single click.

Overview ↥

Quick Export provides a set and forget workflow that reduces the export round-trip time when working on images in Krita for use elsewhere.

Krita's regular export process requires you to confirm that you wish to overwrite the existing file and then to configure export settings. Quick Export lets you configure these things once up front, and from then on your image will be exported immediately.

You can add the Quick Export and Quick Export Configuration action buttons to your toolbar from Settings → Configure Toolbars, and you can assign a shortcut (eg. Alt+E) from Settings → Configure Krita → Keyboard Shortcuts → Menu → File.

FAQ

How do I setup an image for use with this plugin? ↥

First make sure the image is saved. Then press Quick Export in the file menu or toolbar. In the dialog, press '+' to add a config. Now choose an export file type and configure export settings for it. Close the dialog. Now when this is the active image in Krita and you press Quick Export, the image will be exported.

Why does my image not appear in the dialog? ↥

First, have you saved your image as a Krita project file (.kra)? Quick Export ignores unsaved images and files of other types. Second, has it been filtered out? Make sure you are not filtering by name or folder.

Why has my image not exported correctly? ↥

First try exporting again with the same settings using Krita's normal export workflow.

Does Krita export the image wrongly too? See if updating to a later version of Krita helps. If not, consider filing a bug report on Krita's bug tracker.

Does Krita export the image correctly? Try a more recent version of the plugin if available. If not, consider filing a bug report on the plugin's bug tracker. Note that ultimately the plugins export capabilities are constrained by what Krita's API makes available to it. Fixes for some plugin issues may depend on changes to Krita's API.

How can I use the same settings for multiple versions of an image? ↥

Your images should follow the same naming convention as used by Krita's "save incremental version" action.

The version number must always be a series of digits following an underscore after the base name and before the extension. It can not be at the start of the name, in the middle, or after the extension. It can not contain any non-numeric characters.

MyPic.kra, MyPic_0.kra, MyPic_000.kra are examples of acceptable names for the "base" version (or version 0) of the project MyPic.

MyPic_1.kra, MyPic_01.kra, and MyPic_001.kra are examples of acceptable names for version one of that project.

MyPic1.kra or MyPic01.kra are not acceptable names for version one of that project; they would instead be interpreted as the "base" version (or version 0) of the projects MyPic1 and MyPic01, respectively.

Configuration Dialog ↥

The dialog is composed mainly of two parts: the Tree and the Basic Settings Box.

Tree ↥

The Tree contains Folders; the Folders contain Projects; the Projects contain Files.

Folders and Projects can have configurations attached. Files are transient; they just show you which files belong to a Project. You can't select a File, but you can click it to preview its export path.

A Project can be exported by Quick Export without an export config, provided that its Folder has one; in that case it will "inherit" the Folders config. This way you can easily export a whole folder of projects.

Note that unconfigured Folders do not inherit from their folder; only Projects, and only from the Folder they are directly in.

Important: the rows of the tree represent files and folders; moving and removing rows does not move or delete files, and moved and deleted files are not tracked by the tree. If you've moved a project in your file system, you can relocate it in Quick Export by either dragging and dropping its row onto the new folder row (if it's in the tree) or by choosing Relocate from the context menu and navigating to the new folder. Similarly, after renaming a project's files you can relocate it by double-clicking the project name and changing it to match. The same applies to folders.

From left to right, the contents of a row are as follows:

Folder Path or Project/File Name: You can relocate settings for a Folder/Project by editing the path/name.
Add/remove config button: You must add a config before you can configure it. Note that when you remove the config it is lost - this is not an enabled/disabled toggle.
Export settings button: This shows the native Krita export dialog for the chosen export file type so you can configure it.
Open button: Open a folder in the system file manager, or a file in Krita.

The context menu (right-click menu) can contain the following (depending on the selection):

Add Folder/Project: Browse for folders/files to add to the tree. The file dialog starts in the folder of the selected row.
Add/remove unconfigured projects in folder: Add all of the projects contained in this folder to the tree, or removed all of them that don't have an export config.
Show in file browser: Open this folder in the system file manager, same as the open button on the row.
Copy/Paste: You can copy export settings from one row to others. When you press paste, a dialog will ask which settings you want to paste over (name, type, location, scale and export configs as a whole and by type).
Prune export settings: Remove export settings for types other than the currently active type. If for example you added png export settings for a project you are now exporting as jpeg, you can use this option to remove the old png settings. A dialog will appear telling you what will be removed. Note that you aren't required to do this, it just slightly reduces the size of the settings on file.
Remove: Removes this row and its config from the tree. This is different to the remove config button on the row, which leaves the row in the tree. Be aware that if you remove a folder row, the rows of all the projects inside it will also be removed. Remember you can use the revert button to rescue accidental removals.

Above the tree are:

Add Folder/Project button: Browse for folders/files to add to the tree.
Name filter: Filter tree by row path/name.
Folder filter button: Filter tree by folders. Selecting a folder implicitly selects all subfolders by default, but you can deselect them manually.

Basic Settings ↥

Located below the Tree, these are the settings that don't pertain to the export file format.

Basic Settings are broken into three rows: File, Location and Scale. The controls for each are described below from left to right.

File settings row
File name
Project: All version files of this project will export to a file with the project name. For example, puppy_001.kra, puppy_002.kra, puppy_003.kra, etc. will all export to puppy.png (assuming .png type chosen). This means you can make incremental saves of your project without having to adjust your export file name each time.
File: Each version file of this project will export to the same name as the file. For example, cat.kra, cat_002.kra, and cat_003.kra will export to cat.png, cat_002.png, and cat_003.png respectively.
Custom: Enter a custom name. All version files of this project will export to a file with this name. For example, given the custom name "equestrian", horse.kra, horse_002.kra and horse_003.kra will all export to equestrian.png.
File type
Choose the type to export to, such as png, jpeg, exr, etc. Remember to set an export config after choosing a new type. If you can't find the type you're looking for, check if it's hidden (plugin options menu → visible file types).
Location settings row
For an example project with path ~/pictures/animals/dog (that is, project dog in the folder animals, whose files might include ~/pictures/animals/dog.kra):
In same folder: ~/pictures/animals/dog.png
In subfolder: ~/pictures/animals/quadruped/dog.png (given custom subfolder name "quadruped"). You can choose between using the project name or a custom name.
In parent of folder: ~/pictures/dog.png, equivalent to custom location ..
In sibling of folder: ~/pictures/exported/dog.png (given custom folder name "exported"). You can choose between using the project name or a custom name. Equivalent to custom location ../exported
In another folder: You can enter a completely custom path to anywhere on the file system.
The custom path can be relative to the source path (eg. using .. to go up one level, ../../share to go up two levels then down one into a folder named "share", dogpark to go into a subfolder named "dogpark", or leave blank to stay in the same folder).
The custom path can also be absolute. You can use ~ to represent your home folder.
Consult the ? icon's tooltip for more information.
To the right of the custom path field are two buttons: the first switches your path between absolute and relative; the second allows you to browse for a folder.
As you can see, the first four options provide a simple way to locate a folder near to the source, but you can also express any of them as a custom path.
Scale settings row
Optionally, you can scale the image you export.
Scale: Enable or disable scaling.
Side: Which side(s) of the image to scale. You can scale width or height only, or both. You can also select the longest or shortest side.
Size and Units: Set the target size for the chosen side (or if both sides, width first then height). You can specify an absolute size (pixels) or a relative size (percent).
Keep proportions: Maintain aspect ratio. When setting size for one side, will set the size for the other automatically.
Filter strategy: Changes how pixel data is interpolated when scaling. Consult the tooltip in Krita's own scale dialog for more information. Most of the time, auto will suffice.
Beneath the Basic Settings is the export path preview. When the currently selected row in the tree has or inherits an export config, the absolute path it will export to is shown here. Note that project or file names will be substituted with "<ProjectName>" and "<FileName>" when they can't be resolved for the selected row.

Plugin options menu ↥

Located in the bottom-left of the dialog.

Custom icons submenu
Use custom icons: use the plugin's icons or fall back to Krita's own.
Theme options: by default the plugin will try to match the current theme with appropriate light or dark mode icons. If this goes wrong you can manually specify the type of icons to use.
Show export name in file menu: when enabled, the Quick Export action in Krita's file menu will show the file name that will be exported to. Otherwise it will only say "Quick export".
Default export for unsaved images: when enabled, pressing the Quick Export action in the file menu or toolbar for an as-yet unsaved image will perform as though you had pressed the normal export button. Otherwise, the Quick Export configuration dialog will appear.
Load thumbnails & Show thumbnail for selected: if enabled, preview thumbnails will be retrieved from files. If the file doesn't exist or can't be opened, a cross will be shown in the row instead.
Autosave on dialog close: when enabled, any changes you have made in the dialog will be saved when it closes. Otherwise you will be asked to save unsaved changes.
Visible file types submenu: this is a list of all file types that Quick Export can export to. Here you can select a subset of these to show in the file type dropdown.

Saving & Reverting settings ↥

Located in the bottom-right of the dialog are the revert changes and save changes buttons. These will be greyed out if there are no changes to save.

Note that you can interact with Krita as normal while the dialog is open. During this time, any changes made in the dialog will apply immediately. You can, for example, temporarily switch a project's export type to another format, export to it, then revert back to the previous export type.
