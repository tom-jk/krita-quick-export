Quick Export

The Quick Export Plugin allows you to configure export settings for your images once and then export them with a single click.

Overview ↥

If you are working on an image for another project and need to check how it's looking in situ frequently, Krita's native export workflow can slow you down. While it only takes a few seconds each time, these can add up. You may find yourself exporting less often because of the friction involved in doing so, and the result may suffer for it.

The Quick Export plugin provides a set and forget workflow. The first time you go to export your image, you'll first choose where to save it, what type it will be, and the export settings for that type. From then on, your image will export immediately.

Quick Export also supports file versions as produced when you use Krita's save incremental version feature, so you can use the same settings for the duration of a project. For example, the settings for "file.kra" can automatically apply to "file_001.kra", "file_002.kra", and so on.

You can add the Quick Export and Quick Export Configuration action buttons to your toolbar from Settings → Configure Toolbars, and you can assign shortcuts (eg. Alt+E) from Settings → Configure Krita → Keyboard Shortcuts → Menu → File.

Configuration Dialog

How do I setup an image for use with this plugin? ↥

First make sure the image is saved. Then press Quick Export in the file menu or toolbar. In the dialog, choose an export name, type and settings. Close the dialog. Now when this is the active image in Krita and you press Quick Export, the image will be exported.

Why does my image not appear in the dialog? ↥

First, have you saved your image? Quick Export ignores unsaved images.

Second, has it been filtered out? Enable show non-kra and show unopened (and show unstored if in Advanced mode). If the image file isn't currently open in Krita, check that it hasn't been moved to a different folder or name. Open it in Krita then look in the dialog again: is it there?

Perhaps its export settings had not been saved in the plugin. Remember to change one of its settings or export it with the export now button, or check the store settings checkbox directly (if in Advanced mode). If that checkbox isn't checked, the plugin will not save settings for this image (and will forget settings already saved, if any). By default, if the row is highlighted in blue, it will be saved.

Why has my image not exported correctly? ↥

If you've double checked your export settings and the image still comes out wrong, you should first try exporting again with the same settings using Krita's normal export workflow.

Does Krita export the image wrongly too? If so, that may be a bug in Krita. See if updating to a later version of Krita helps. If not, consider filing a bug report on Krita's bug tracker.

Does Krita export the image correctly? If so, that may be a bug in the plugin. Check if a more recent version of the plugin is available. If not, consider filing a bug report on the plugins bug tracker.

Note that ultimately the plugins export capabilities are constrained by what Krita's API makes available to it. Fixes for some plugin issues may depend on changes to Krita's API.

How can I use the same settings for multiple versions of an image? ↥

In the configuration dialog, on that image's row, set versions options to "this and later versions of this file". See versions options.

To work, your images should follow the same naming convention as used by Krita's "save incremental version" action.

The version number must always be a series of digits following an underscore after the base name and before the extension. It can not be at the start of the name, in the middle, or after the extension. It can not contain any non-numeric characters.

"MyPic.kra", "MyPic_0.kra", "MyPic_000.kra" are examples of acceptable names for the "base" version (or version 0) of the image "MyPic".

"MyPic_1.kra", "MyPic_01.kra", and "MyPic_001.kra" are examples of acceptable names for version one of that image.

"MyPic1.kra" or "MyPic01.kra" are not acceptable names for version one of that image; they would instead be interpreted as the "base" version (or version 0) of the images "MyPic1" and "MyPic01", respectively.

Why does my image not have the settings I previously set in the dialog? ↥

You might be looking at the wrong row. Every currently open document has a row of its own in the dialog, but in addition to that, a document that is a version of an image will "inherit" the settings of that image (provided that image's settings are set to apply to its versions).

For example: settings are stored for "Image.kra", and its versioning options are set to "versions of this file". Now we open the dialog for "Image_001.kra". There will be a row for "Image_001.kra", but the settings that are actually used when you export the document from the file menu or toolbar are the settings for "Image.kra", because it it set to apply to versions of it, and "Image_001.kra" is one. So to change the export settings for "Image_001.kra", you should change the export settings for "Image.kra".

Configuration Dialog

Settings rows ↥

Each row in the dialog represents a file, one for each file currently open in Krita, plus those not currently open but for which export settings have previously been saved.

When you open the dialog, it will scroll to show the row with the settings that apply to the active document.

From left to right, the contents of a row are as follows:

Store settings checkbox
Indicates that these settings will be saved. If you no longer need to export an image, you can uncheck this and Quick Export will forget about it. Remember that this won't happen until you save your changes. (Advanced mode only.)
Open file button
Press this button to open a currently unopened file. An asterisk will show here for the file that was active in Krita when the dialog opened. Otherwise this space is blank.
Source file path and name
The folder and file name with extension for the image to export from.
Versions options
Here you can choose which files these settings apply to.
This exact file
Settings apply to the specified source file alone. Versions of this image are ignored.
Versions of this file
Settings apply to the specified source file and any file in the same directory with the same name plus an underscore and one or more digits (and the same file extension). For example, settings for "file.kra" will also apply to "file_001.kra", "file_002.kra", etc. If the source file name already has a version number (even if it's "_000"), you probably want to use the next option; "file_000.kra" would be considered the base for version files named "file_000_001.kra", "file_000_002.kra", etc.
This and later versions of this file
Settings apply to the specified source file and any version of that file in the same directory with a greater version number. For example, settings for "file_005.kra" will also apply to "file_006.kra" and "file_007.kra", but not to "file_004.kra" or "file.kra". Use this option if you name the first version of your image with a version number, or wish to change export settings part way through a project but also keep the old settings for earlier versions.
Most of the time, use option 3 (this and later versions).
Press the arrow button to show a list of all the files that the settings on this row apply to.
Export file path button
Click this to open a file dialog where you can choose the folder and name to export to. When you are exporting to the same folder as the source, this will show as a dot.
Export file name
You can also set the name to export to here. Note that when the settings on this row apply to multiple versions of the image, exporting from any of them will overwrite this file. That is, exported files are unversioned (eg. "file.kra", "file_001.kra", "file_002.kra" etc. will all export to "file.png", not "file.png", "file_001.png", "file_002.png", etc.)
Export file type
This dropdown lists all supported file types. Select which type you wish to export to. Note that by default, only the most commonly used types are shown; see Visible file types.
Export settings
The settings shown here depends on the export file type you've chosen. Each setting has a tooltip containing information about what it does.
Export now button
You can export an image at any time directly from the dialog (provided it is open in Krita). For rows applying to multiple currently open images, this button shows a menu where you can select the one to export.

Dialog appearance options ↥

Below the list are options for changing its appearance.

Show non-kra/unopened/unstored
Filters the list.
Non-kra: Typically you will work in and export from Krita project files (.kra), so this option lets you hide images of other types.
Unopened: By default the list will contain every file for which export settings are stored, including those not currently open in Krita. This option lets you hide those.
Unstored (Advanced mode only): By default the list will contain every file currently opened in Krita (excluding unsaved images), including those for which no export settings are currently stored. Once you have set up export settings for the images you want to export, you can disable this option to reduce clutter.
Display mode
This dropdown lets you pick a display mode for this list.
Minimized: Every row will be as small as possible. Settings can be accessed as a popup by pressing the button in the settings column.
Compact: Every row will be as short as it can be while still showing all settings for the chosen image type. Settings will appear as buttons without labels, but you can still see information about them in their tooltips.
Focused: Every row will be compact or minimized, except for a "focused" row which will appear expanded. By default, the image that was active when the dialog opened will be focused. You can change focus by double-clicking anywhere on a row.
Expanded: Every row will be as tall as necessary to show their settings, each on their own line with a label.
Minimize unfocused
Only appears in focused display mode. When enabled, unfocused rows will be minimized, as in minimized display mode except the settings are hidden outright. When disabled, unfocused rows will be compact.
Fade sliders
These sliders let you fine tune the intensity of some visual effects. Press the "Fade..." button to show them, and again to hide.
Contrast: adjust the intensity of the alternating row colours.
Settings: adjust the opacity of settings for unhovered rows. This applies only to compact and expanded display modes. Fading unhovered settings can make the list feel less cluttered.
Stored: adjust the intensity of the row highlight for settings that will be saved (that is, rows where the store settings checkbox is checked).

Dialog behaviour options ↥

The second line of options adjust the dialogs behaviour.

Advanced mode
Advanced mode allows you to more finely adjust how and when export settings are saved.
When turned off:
The store settings checkbox column will be hidden
The show unstored checkbox will be checked and hidden
The store on modify checkbox will be checked and hidden
The store on export checkbox will be checked and hidden
The autosave on close checkbox will be checked and hidden
The save settings button will be hidden
Essentially, export settings will be saved for any row you interact with automatically.
When turned on, these will be visible and return to whatever you last set them to. You only need to turn advanced mode on if you want to remove export settings for images from Quick Export.
Store on modify/export
When enabled, a row's store settings checkbox will be automatically checked:
Modify: if you change any of its settings (the checkbox will not be unchecked if you return them to their original state).
Export: if you export it with the export now button.
Autosave on close
When enabled, any changes you have made in the dialog will be saved when it closes. Otherwise you will be asked to save unsaved changes.
Save settings
Click this button to save changes made in the dialog. The button will be greyed out if there are no changes to save.

Plugin options menu ↥

Finally, there is the Quick Export plugin options menu button on the left side of the status bar.

Custom icons submenu
Use custom icons: enable or disable the use of the plugin's icons. When disabled the icons will fall back to some close approximations from Krita's own icon set.
Theme options: by default the plugin will try to match the current theme with appropriate light or dark mode icons. If this goes wrong you can manually specify the type of icons to use.
Show export name in file menu
When enabled, the Quick Export action in Krita's file menu will show the file name that will be exported to. Otherwise it will only say "Quick export".
Default export for unsaved images
When enabled, pressing the Quick Export action in the file menu or toolbar for an as-yet unsaved image will instead perform a normal Krita export, that is, it will behave as though you had pressed the normal export button.
Otherwise, the Quick Export configuration dialog will appear.
Show thumbnails for unopened images
If enabled, the dialog will retrieve preview thumbnails from stored but unopened files. If the file doesn't exist or can't be opened, a cross will be shown instead. If disabled, those rows will show no thumbnail until they are opened in Krita.
Visible file types submenu
This is a list of all file types that Quick Export can export to. Here you can select a subset of these to show in the file type dropdown on each export settings row. By default, only .jpg, .jpeg and .png are enabled, so these will be the only ones you can export to.
If you wish to export to .gif, for example, first enable it in this menu, then you can select it from the export file type dropdown for your image.
Hiding an image type does not affect any image already exporting to that type.
Wider grabber for resizing columns
This is an accessibility tweak. When enabled, the region you must click inside in order to resize columns will be twice as wide. Enable this if you frequently sort or reorder columns by accident when trying to resize them.
